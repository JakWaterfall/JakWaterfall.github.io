Git Hack.

Git Hack was an event suggested to me by DevSoc whom I had joined a few days prior. Git hack was an all-day video conference which was designed to teach an amateur how to use Git and GitHub professionally. I learned all about using Git and GitHub for version control and how to collaborate with others, allowing me to work on a project simultaneously with more people while not interrupting others or my own work. Some concepts I learnt are:

  • Creating repositories for a project.
  • Cloning (downloading) repos onto my computer.
  • Making commits and pushing changes to which ever branch Iâ€™m working on.
  • Creating branches to allow me to work on a copy of the project without fear of breaking anything.
  • Merging said branch back into the master branch to make by changes or add my feature I was working on to the main copy of the project.
  • Making pull requests to propose your changes / feature to the owner of a project I have been working on.
  • Using GitHub as a social platform / following your favourite creators.
  • Using GitHub as a platform to show off your skills and interest future employers.
